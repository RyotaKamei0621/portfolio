システム構成 — 5段階パイプライン

①Tailscaleによるポート開放なしの安全な通信 → ②WebクライアントからRaspberry Piへコメント送信 → ③Liquid AI LFM2.5（1.2Bパラメーターモデル）/ llama.cpp によるCPU推論で応答生成 → ④VOICEVOX Engine（ローカルREST API通信）で音声合成・完全ローカル＆オフライン動作 → ⑤PixiJS + Live2D Cubism SDK によるクライアントサイド（スマホ側）WebGLレンダリングでモデル表示・口パク同期

描画・アニメーション処理をすべてスマホ（クライアント）のGPU/CPUにオフロードすることで、非力なRaspberry Pi側の負荷を劇的に軽減する設計にした。ラズパイはテキスト・音声データをスマホに渡すだけで、重い描画処理には関与しない。

表情認識ゲームプレイシステムは、顔の動きをゲーム操作コマンドに変換し、サーボモーターで物理ボタンを押下するインターフェース。ゲーム機本体に電気的な改造を加えず、外部から物理的にボタンを押すことで操作する点が特徴。

システム構成

カメラで顔画像を取得 → OpenCVで口の開閉・首の傾きを検出 → 操作コマンドに変換 → Raspberry Piからサーボモーターを制御してDSの物理ボタンを押下。

首の左右傾き → 十字キー：両目を結ぶ線の角度が一定以上傾いた場合に左右入力として扱う。

2026/1

Role

Processingで制作した3Dアクションゲーム。マウスを回す操作をアンカーの巻き取り速度に変換し、ドラッグの速度と軌跡を斬撃判定に反映することで、立体機動らしい身体的な操作感を再現した。

デモ映像

立体機動訓練シミュレーター ショートデモ

コアインタラクション設計

① マウス回転 → 接近速度：フレームごとのマウス角度変化量を取得し、その絶対値をアンカー巻き取り速度として加算。マウスを速く回すほど高速で巨人に接近する、フィジカルな操作感を実現した。

② マウス軌跡 → 斬撃判定：マウスドラッグの軌跡から移動距離を計算し、閾値以上の速度でうなじ付近を通過した場合のみ有効打と判定。単なるクリックではなく「振り抜く」動作が必要な設計にした。

③ 近づき過ぎたときのノックバック：巨人との距離が一定以下になると、強制ノックバック・タイム減少・レッドフラッシュを同時実行。巨人に接近しすぎないよう、プレイヤーへのリスクとして機能する。

// ① マウス回転 → 巻き取り速度に変換 void detectRotationAndMove() { float currentAngle = atan2(mouseY - cy, mouseX - cx); float delta = currentAngle - lastMouseAngle; playerDist -= rotationInput * ANCHOR_SPEED; } // ② マウス軌跡 → 斬撃ダメージ計算 void updateBladeTrail() { PVector s = mouseTrail.get(0); PVector e = mouseTrail.get(mouseTrail.size()-1); float d = PVector.dist(s, e); float damage = (distToTitan < 150) ? d * 2.5 : 0; } // ③ 近づき過ぎ → ペナルティ処理 void applyPenalty() { playerDist = 1000; timeLeft -= 3.0; isAttached = false; flashColor = color(255, 0, 0, 150); }

チュートリアル画面の進行

ゲーム開始から操作方法の提示、アンカー発射、巻き取り、接近、斬撃に至るまでのフローをcase0〜case4の画面で説明。各caseでプレイヤーへの指示内容が変わり、段階的に立体機動操作を習得できる構成にした。

case0：視点移動 → case1：アンカー発射 → case2：巻き取り → case3：斬撃 → case4：チュートリアル終了

ランク評価システム

PicScatter — 画像分割・ランダムシャッフルアプリ

2025/8

Role

企画・実装
GUI設計・Booth公開

Tools

Python / Tkinter / Pillow

選択した画像を縦横に分割し、各ブロックをランダムに再配置するPython製GUIアプリ。脳波実験のオッドボール課題で使う視覚刺激の作成が目的。画像選択・分割数指定・seed値での再現性確保・プレビュー・保存をGUI上で完結できるよう設計し、Boothで公開・販売した。

使用イメージ

画像を選択し、分割数とseed値を指定してボタンを押すだけで処理が完了。プレビューで確認しながら保存できるシンプルな操作フローを実現した。

画像選択 → 分割数指定 → シャッフル → プレビュー → 保存の一連の操作

GUI 設計 — Tkinter + Pillow

Tkinterで画像選択・分割数・seed値・補正方式・プレビュー・保存の操作UIを構築。Pillowで画像読み込み・サイズ補正・ブロック分割・再配置・PNG/JPEG保存を実装。6ステップのワークフローをGUI上で完結。

出力バリエーション — 分割数による表現の変化

3×3 / 10×10 / 10×20 / 30×30 / 50×50 の分割バリエーション

ユースケース — 脳波実験 ＆ ゲーム演出への活用

脳波実験（オッドボール課題）では、高頻度の標準刺激と低頻度の標的刺激を混在させてERP（P300成分）を測定。標的刺激の画像全体の輝度・色彩などの特性を保ったまま、認知対象だけを破壊した対照刺激を用意することが可能。また、ホラーゲーム「研究事件」で登場する、パソコンが急に砂嵐画面になる場面では、黒/白/ピンク/緑/黄などを配置した数パターンの画像に対してPicScatter処理を行うことで対応した。
